stardog-admin db migrate Manual Page

NAME


stardog-admin db migrate —Migrates a database to the latest index format.

SYNOPSIS


stardog-admin [ --server <server url> ]db migrate [ {-p | --passwd} <password> ] [ {-P | --ask-password} ] [ {-u | --username} <username> ] [ {-v | --verbose} ] [--] <database>

OPTIONS


-p <password>, --passwd <password>
Password.
-P, --ask-password
Prompt for password.
--server <server url>
URL of Stardog Server. If this option isn't specified, it will be read from JVM argument 'stardog.default.cli.server'. If the JVM arg isn't set, the default value 'snarl://localhost:5820' is used. If server URL has no explicit port value, the default port value '5820' is used. Example: 'stardog --server snarl://12.34.56.78:5820 server stop'
-u <username>, --username <username>
User name.
-v, --verbose
Verbose flag that causes more detailed information to be printed
--
This option can be used to separate command-line options from the list of argument, (useful when arguments might be mistaken for command-line options
<database>
The name of the database for the operation

DISCUSSION


Migrates a database using a deprecated on-disk index format to the current format. If the database already uses the latest format, contents will not be migrated. Always backup databases before migrating.

EXAMPLES


Migrate the database 'foo':

    $ stardog-admin db migrate foo
